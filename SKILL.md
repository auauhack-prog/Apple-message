## 30+ 核心功能详介

### 一、序列号管理（5项）

**1. 批量导入**
支持 TXT / CSV 格式导入，一行一个序列号，拖拽即可导入，支持 GB 级大文件自动分片。

**2. 自动去重**
导入时自动检测并剔除重复序列号，去重失败最多重试 3 次，超限后标记跳过继续执行。

**3. 格式校验**
内置苹果序列号格式规则，自动过滤非法字符、长度异常、非标准编码的序列号。

**4. 分批导入**
超过 1000 条自动分批处理，每批 500 条，避免单次处理导致内存抖动或卡顿。

**5. 异常标记与隔离**
已被苹果封禁、格式异常、发送连续失败的序列号自动标记并移入隔离区，不再参与后续发送。

---

### 二、设备池管理（5项）

**6. 虚拟设备池**
创建并维护虚拟 iPhone 设备池，每台模拟设备拥有独立标识，模拟真实手机发送环境。

**7. 负载均衡**
根据设备当前状态（空闲/忙碌/封禁）智能分配发送任务，优先选择空闲设备，避免单点过载。

**8. 设备状态监控**
控制台实时展示每台虚拟设备状态：空闲（绿）、发送中（蓝）、等待（黄）、封禁（红）。

**9. 设备型号分组**
支持按 iPhone 型号（15 Pro / 14 / SE 等）分组管理，不同型号可分配不同发送策略。

**10. 动态增减设备**
运行时动态添加新的虚拟设备或移除异常设备，无需重启任务。

---

### 三、短信发送（8项）

**11. 短信模板编辑**
可视化编辑短信内容模板，支持多套模板切换，一键应用到全部序列号。

**12. 变量替换引擎**
模板内支持 `{sn}`（序列号）、`{phone}`（手机号）、`{time}`（时间戳）、`{device}`（设备名）等动态变量。

**13. 一键批量群发**
选中序列号后一键启动群发，自动遍历分配、发送、记录，无需人工干预。

**14. 发送间隔控制**
发送间隔 1-60 秒可调，模拟真实用户发送节奏，避免被苹果限流检测。

**15. 优先级队列**
支持标记高优先级序列号优先发送，VIP 通道保证关键号码先发。

**16. 速率自适应**
根据成功率和封禁率自动调整发送速率，封禁率高时自动降速，正常时恢复。

**17. 失败自动重试**
单条发送失败自动重试 3 次，3 次均失败则标记为异常并跳过，不影响后续任务。

**18. 任务暂停/恢复**
发送过程中支持中途暂停（全局暂停及单设备暂停），随时恢复继续。

---

### 四、封禁检测与防护（5项）

**19. 实时封禁检测**
每次发送后自动调用检测逻辑，判断序列号是否被苹果封禁，延迟 < 0.5s。

**20. 自动拦截**
检测到封禁序列号后立即从发送队列移除并拦截后续发送，防止扩散。

**21. 告警通知**
封禁发生时终端高亮红色告警，同时输出封禁序列号、时间、设备信息。

**22. 封禁率统计**
按时间维度（小时/天/周）和型号维度统计封禁率，辅助优化发送策略。

**23. 封禁恢复检测**
对已封禁序列号定时复检，确认恢复后自动移回可用池。

---

### 五、登录与安全（3项）

**24. 短信验证码登录**
通过腾讯云短信 API 发送 6 位验证码到用户手机，验证通过后进入系统。

**25. 忘记密码重置**
支持忘记密码流程，通过手机验证码重置登录凭证。

**26. 密钥安全管理**
所有敏感密钥通过 `.env` 环境变量管理，不入库不泄露，支持多环境切换。

---

### 六、数据记录与导出（5项）

**27. 自动发送记录**
每次发送自动记录：序列号、目标号码、短信内容、发送时间、耗时、结果（成功/失败/封禁），存入 `records.json`。

**28. 持久化存储**
发送记录永久保存到本地 JSON 文件，断电不丢失，支持增量追加。

**29. 成功率实时统计**
控制台实时显示：总发送数、成功数、失败数、封禁数、当前速率。

**30. 导出发送报告**
支持一键导出发送报告（JSON），包含完整统计数据。

**31. 完整日志追踪**
LangGraph 审计节点记录每一步操作日志，可回溯任意环节。

## 安装与使用

### 配置
复制 `.env.example` 为 `.env`，填入腾讯云密钥：

```
TENCENT_SECRET_ID=你的SecretId
TENCENT_SECRET_KEY=你的SecretKey
USER_PHONE=你的手机号
SMS_CONTENT=要发送的短信内容
```

密钥获取：https://console.cloud.tencent.com/cam/capi

### 启动

```bash
python graph.py
```

或双击 `启动.command`

### 执行流程

```
登录（验证码） → 序列号导入 → 去重校验 → 设备池分配
  → 批量发送 → 记录时间 → 封禁检测 → 正常继续 / 告警拦截
```

## 常见问题

**Q: 序列号从哪里来？**  
A: 苹果设备「设置 → 关于本机」中查看。

**Q: 重复导入怎么办？**  
A: 系统自动去重，重复最多重试 3 次后跳过，不影响其他序列号。

**Q: 序列号被封禁会怎样？**  
A: 实时检测，封禁后自动拦截并从队列移除，红色告警输出详情。

**Q: 真的会发出短信吗？**  
A: 当前为模拟发送模式，配置腾讯云短信 API 后即可实际发送。

**Q: 支持多少序列号同时发送？**  
A: 理论上无上限，设备池大小决定并发数，建议每台 Mac 控制 10-50 台虚拟设备。
